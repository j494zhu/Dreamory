## ★ 核心命题(北极星)

- `[原则]` **★ LLM 要有反向提供情绪价值的链路。** 核心问题:llm 拟人的真实性反而是次要的,真正核心是让用户感觉到满足;为了这种满足感,绝对不应该去迁就一种"显示感觉"。现在踏马的怎么全是**用户在给 llm 提供情绪价值**(bid、commitment 之类)?llm 怎么没有给用户提供情绪价值的链路?
  - 相关落点:`↔` 依恋理论双向接投标/主动安慰(架构) · `↔` affection 更多上涨路径 + 印象表(好感度) · `↔` narrative 不处理正面归因(架构) · `↔` 用户不能带情绪、只能当公主供着(关系事件) · `↔` 高情绪张力时长限制(架构)。

- `[原则]` **★ 不要长时间保持过度浪漫。**("小别胜新婚")sonnet 指出女方 thinking 暴露了"策略性回应"(先接浪漫→引出话题种子→轻推互动),如果无限放大,用户会习惯"每次都被完美回应",这和真实关系有本质差别——真实关系里"被误解/被冷落一下/对方那天没接住梗"恰恰是真实感与张力的来源。一直过高的情绪张力反而让人失去"命定时刻"的致命浪漫感。**或许加一个高情绪的持续时间?** 估计只有几分钟对这种浪漫有特别高的接受度。系统可能需要进一步重新设计。
  - 后续:吃透本阶段后,是时候看看 Claude Code 的设计,可能需要 skills / claude.md 这种结构——先把抽象基本规则(如"不要过长时间保持过度浪漫")写下来,再用 skills 具体应对。

- `[原则]` **AI 伴侣的标准**:在不让人反感的范围内,可以表现得和真人不一样(比如说记性很好、重承诺),但**不可以鸡毛蒜皮的小事都记得要拉出来说**。 `↔` _escalate_old_loops · `↔` _render_memory 每条都回应所有 openloop。
